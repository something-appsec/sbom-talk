# Security Policy

## Supported Versions

Since we don't maintain dedicated release versions of our talk, support is only provided for the latest version as found in the [`main` branch](https://github.com/something-appsec/sbom-talk/tree/main).

## Reporting a Vulnerability

In case you identify a security vulnerability in our repository, please either file an issue on our [Issue Board](https://github.com/something-appsec/sbom-talk/issues) or, if possible, propose a Pull Request.

We will try to come back to any raised issues within a matter of days (usually in course of the following weekend).

If your vulnerability is accepted, we will try to implement corresponding mitigations.
